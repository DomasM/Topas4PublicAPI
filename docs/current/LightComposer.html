Preliminary

Introduction

LightComposer is an end-user-oriented application for controlling OPA-based laser sources produced by Light Conversion. Interactive laser system control via the LightComposer GUI is described in the User Manual of a particular system, this document covers local and remote control via the LightComposer API.

LightComposer provides a GUI for interactive laser system control and an API service for remote control, automation and advanced applications. LightComposer operates on top of Topas4, whose server component carries out the actual hardware control. Depending on system configuration, one or more Topas4 server instances are used for hardware control and all must be running for the laser system to function properly. Running LightComposer should start all the necessary server instances. From the LightComposer user standpoint, communication occurs between the client application and the LightComposer app. All the necessary backend operations are handled by LightComposer directly.

API Control Basics

API control is provided via a REST interface. API calls between the client and LightComposer application are sessionless, i.e., there are no handles, control locks or opening/closing of devices. Multiple clients can use the REST service at the same time and LightComposer GUI can be used at the same time too.

The remote application sends HTTP REST requests to LightComposer application. A response is sent back to the remote application as part of a HTTP exchange. The contents of REST messages are serialized using JSON.

Since REST commands are sessionless they are non-blocking. As a result, commands that take some time to finish (e.g. set wavelength, set GDD, open shutter) are only initiated by the REST command. You can track the progress of these actions by repeatedly polling their status.

Detailed description of all sub-services

  • Main control

Set wavelength, GDD, control shutters, get current device status.

  • Output beam steering

Advanced methods to control output beam steering and stabilization

  • Pump Laser

Methods to control pump laser for OPA: turn on, standby, set pulse picker diver etc

Postman collection

A very quick and convievent way to start playing with API is to import provided Postman collection. Set correct collection environment variable “url” in Postman after you import collection to start using it with your device. It should look like http://127.0.0.1:35120/CTP21456/v0/ where CTP21456 is replaced with serial number of your device, and correct IP address and port is supplied too.
